# 创建空应⽤

## AI创建空应用入口

**功能说明**：平台提供多种便捷入口快速进入AI创建空应用流程，用户可根据当前所在位置选择最便捷的方式。

**方式一：首页快速创建入口**

1. 登录智能数字员工平台后，默认进入首页
2. 在首页中间区域找到"快速创建入口"模块
3. 点击【AI创建空应用】快捷按钮
4. 直接跳转到AI创建空应用页面，开始填写应用基础信息

**适用场景**：用户刚登录系统，希望快速开始创建新应用

![image-20260122180229584](https://raw.githubusercontent.com/pinganki37-arch/Intelligent-Digital-Employee-Platform/main/assets/image-20260122180229584.png)

**方式二：我的应用页面创建**

1. 点击导航栏的【我的应用】进入应用管理页面
2. 在应用列表页面点击【新建应用】按钮
3. 在弹出的创建方式选择菜单中，选择【AI创建空应用】
4. 进入AI创建空应用页面

**适用场景**：用户在管理现有应用时，需要创建新的应用

![image-20260122180255065](https://raw.githubusercontent.com/pinganki37-arch/Intelligent-Digital-Employee-Platform/main/assets/image-20260122180255065.png)

**方式三：通过AI智能助手创建**

1. 在平台任意页面，点击右下角的AI智能对话助手图标
2. 在对话框中输入创建应用的需求，如"我想创建一个新的空应用"
3. AI助手理解需求后，会引导用户进入AI创建空应用流程
4. 也可以直接在对话中完成应用基础信息的填写，AI助手会自动触发创建流程

**适用场景**：用户习惯通过对话式交互完成操作，或不确定如何开始创建应用

![image-20260122180357916](https://raw.githubusercontent.com/pinganki37-arch/Intelligent-Digital-Employee-Platform/main/assets/image-20260122180357916.png)

<div style="border: 1px solid #00CC66; background-color: #F2FFF9; padding: 12px; border-radius: 4px; margin-top: 10px;"> <strong>⏰注意事项：</strong> <br>推荐使用首页快速入口：操作最简便，步骤最少<br>
AI助手智能引导：适合新手用户，AI会提供全程指导<br>
所有入口效果相同：无论从哪个入口进入，后续的创建流程完全一致 <br></div>

## 功能概述

本功能旨在通过AI辅助实现应用的快速创建，用户仅需简单输入应用描述，即可由AI自动生成应用基本信息、配置数据源，并完成应用的初始化创建，最终进入低代码设计器进行后续开发。该功能大幅降低了应用创建的技术门槛，提升了开发效率，适用于需要快速搭建应用框架的场景。

## 创建空应用流程总览

<div style="border: 1px solid #0088CC; background-color: #F0F8FF; padding: 12px; border-radius: 4px; margin-top: 10px;"> <strong>流程步骤：</strong>  基本信息配置 → 数据源配置 → 确认创建 → 应⽤创建中 → 应用构建页面 </div>

### 应用基本信息配置

**功能定位：**收集应用核心描述信息，由AI根据应用描述自动生成相应的应用名称和编码。

![image-20260122180622681](https://raw.githubusercontent.com/pinganki37-arch/Intelligent-Digital-Employee-Platform/main/assets/image-20260122180622681.png)	

**核心操作步骤：**

1. **输入应用描述**：在"应用描述输入框"中简要描述应用用途和功能（如"人才管理系统，用于管理人员简历、面试记录和入职流程"）。
2. **一键AI生成**：点击"一键AI生成"按钮，平台实时根据生成应用名称和编码。
3. 调整生成结果：
   - **应用名称**：支持手动修改，为应用的中文显示名称。
   - **应用编码**：支持手动修改，为应用的唯一标识（需满足小写字母、数字和中划线组成，以字母开头，最多17个字符）。
4. **下一步**：基本信息确认无误后，点击"下一步"按钮。

**应用命名规范建议：**

**✅ 推荐格式：**
应用名称：使用中文，简洁明了，3-10个字。
应用编码：使用英文小写 + 中划线，如 `user-management`。

**❌ 不推荐格式：**
应用名称：过长或包含特殊符号。
应用编码：使用大写字母、下划线或中文。

| 类型     | 应用名称             | 应用编码        | 说明                           |
| :------- | :------------------- | :-------------- | :----------------------------- |
| ✅ 推荐   | 员工自助系统         | self-service-hr | 清晰易懂，编码规范             |
| ✅ 推荐   | 办公空间租赁         | office-rent     | 格式规范                       |
| ❌ 不推荐 | 系统123              | System_Test     | 命名不清晰，编码含大写和下划线 |
| ❌ 不推荐 | 员工自助管理系统V2.0 | SelfService     | 名称过长，编码使用驼峰         |

### 应用数据源信息配置

功能定位：配置应用所需的数据库连接信息，AI辅助生成数据源名称。在该处配置的数据源信息后续用于应用内表单数据模型表创建以及后续页面数据存储。

![image-20260122180528937](https://raw.githubusercontent.com/pinganki37-arch/Intelligent-Digital-Employee-Platform/main/assets/image-20260122180528937.png)

**配置项说明：**

- **选择数据源：**点击“选择数据源”可选择到本人历史连接的数据源，选择后可直接获取数据源信息到相应输入框中。
- **数据源类型**：默认polarDB，支持MySQL、Oracle、SQL Server等。
- **连接地址**：数据库连接URL（如 jdbc:mysql://XXX/aigent）。
- **用户名/密码**：数据库访问凭证。
- **数据源名称**：点击"AI生成"，系统自动生成具有业务含义的名称。

**操作步骤：**

1. 填写完整的数据库连接信息。
2. 点击"连接测试"按钮，验证连接是否正常。
3. 测试通过后，配置连接池参数。
4. 点击"下一步"按钮。

### 确认创建

**功能定位：**展示用户在之前步骤中配置的所有信息，供用户最终确认。

![image-20260122180707784](https://raw.githubusercontent.com/pinganki37-arch/Intelligent-Digital-Employee-Platform/main/assets/image-20260122180707784.png)

**核心能力：**

- **基本信息区域**：汇总展示应用名称、应用编码、应用描述。
- **数据源配置区域**：汇总展示数据源类型、连接地址、数据源名称。

**操作：**确认信息无误后，点击"创建"按钮触发应用创建流程。

### 应用创建中

系统进入加载状态，展示创建进度。依次展示"正在创建aPaaS应用"、"应用创建成功"、"正在创建数据源"、"数据源创建成功"等提示。所有步骤完成后，自动跳转至低代码设计器。

![image-20260122180913229](https://raw.githubusercontent.com/pinganki37-arch/Intelligent-Digital-Employee-Platform/main/assets/image-20260122180913229.png)

### 应用构建页面

应用创建完成后进入的主界面，提供应用后续开发的操作界面。

- **快速创建页面入口**：包含Excel创建表单、AI创建表单、简单报表、复杂报表等创建方式。
- **AI助手**：右侧AI助手，可根据用户需求分析业务、生成页面设计方案。
