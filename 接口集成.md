# 接口集成

## 入口

点击顶部导航栏【接口集成】（仅平台管理员可见）。

![image-20260122162113396](https://raw.githubusercontent.com/pinganki37-arch/Intelligent-Digital-Employee-Platform/main/assets/image-20260122162113396.png)

## 功能概述

接⼝集成模块是智能数字员⼯平台的"服务连接中枢",负责第三⽅系统接⼝的 管理、 调⽤和监控。 模块提供完整的接⼝⽣命周期管理能⼒,包括接⼝注册、 测试、 监控、 ⽂档管理等功能。

<div style="border: 1px solid #00CC66; background-color: #F2FFF9; padding: 12px; border-radius: 4px; margin-top: 10px;"> <strong>✅ 本模块是什么：</strong> <br>本模块是一个<strong>接口信息存储和调用记录查询</strong>模块。它负责维护接口的配置信息（如URL、请求方式、Header等），并接收业务系统上报的调用日志供用户查询。<br></div>

<div style="border: 1px solid #FF0239; background-color: #FAF1F3; padding: 12px; border-radius: 4px; margin-top: 10px;"> <strong>❌ 本模块不是什么：</strong> <br>本模块不直接调用接口，也不是API网关。它不负责实际的数据转发或业务逻辑执行，不提供在线连接外部系统的通道。<br></div>

## 接口注册管理

### 注册新接口

注册接口是管理接口信息的第一步。

1. 在【接口注册】页面，点击右上角的**【+ 注册接口】**按钮。
2. 在弹出的对话框中，填写接口详细信息。

![image-20260123135357433](https://raw.githubusercontent.com/pinganki37-arch/Intelligent-Digital-Employee-Platform/main/assets/image-20260123135357433.png)

**接口信息字段说明：**

| 字段名称 | 必填 |                          说明                          |
| :------: | :--: | :----------------------------------------------------: |
| 接口名称 |  是  |      输入接口的名称标识，建议使用清晰的中文名称。      |
| 接口分类 |  是  |      从下拉菜单中选择接口所属分类（默认：其他）。      |
| 请求方法 |  是  |    选择HTTP请求方法：GET / POST / PUT / DELETE 等。    |
| 接口地址 |  是  | 输入完整的接口URL地址（以 http:// 或 https:// 开头）。 |
| 接口编码 |  否  |         接口的唯一标识码，留空则系统自动生成。         |
| 接口描述 |  否  |           填写接口用途、功能说明及注意事项。           |
|  提供商  |  否  |       填写接口提供方信息（如：第三方系统名称）。       |
| 接口状态 |  否  |           选择：启用 / 禁用（默认：启用）。            |
|  版本号  |  否  |                接口版本，默认为 1.0.0。                |
| 超时时间 |  否  |        建议设置接口超时时间（秒），默认 30秒。         |

### 配置请求头（Headers）

如果接口需要特定的Header（如认证信息、内容类型），请按以下步骤配置：

1. 在注册接口对话框下方找到**【请求头配置】**区域。
2. 点击**【+ 添加请求头】**按钮。
3. 填写请求头信息：
   - **KEY：** 请求头的键名（例如：Content-Type, Authorization）。
   - **VALUE：** 请求头的值（例如：application/json, Bearer token）。
   - **DESCRIPTION：** 请求头说明（可选）。
4. 如需添加多个请求头，重复步骤2-3。
5. 点击**【确定】**按钮保存接口注册信息。

### 配置验证测试

**ℹ️ 说明：** 此处的测试功能仅用于**验证接口配置的正确性**（如URL是否可通、参数是否正确），并非实际业务场景的自动化调用。

![image-20260123135434932](https://raw.githubusercontent.com/pinganki37-arch/Intelligent-Digital-Employee-Platform/main/assets/image-20260123135434932.png)

1. 在接口列表中找到刚注册的接口。
2. 点击该接口操作列的**【测试】**按钮。
3. 在弹出的【接口测试】对话框中：
   - **请求头：** 确认已配置的Header信息。
   - **查询参数：** 如需添加Query参数，在此页签配置。
   - **请求体：** 如果是POST请求，在此页签填写JSON或其他格式的内容。
4. 点击右下角**【执行测试】**按钮。
5. 查看右侧测试结果：
   - **HTTP状态码：** 200表示成功。
   - **响应时间：** 接口响应耗时。
   - **响应内容：** 接口返回的实际数据。

### 接口管理操作

在接口列表页，您可以对已注册的接口进行管理：

- **编辑：** 点击操作列的【编辑】图标，修改接口信息。
- **删除：** 点击操作列的【删除】图标，移除不再使用的接口。
- **监控：**点击操作列的【监控】图标，可直接跳转至【调用记录】页面查看该接口调用记录。
- **下线：**点击“激活”状态接口操作列的【下线】图标，可将接口下线，下线后平台不再记录该接口的调用记录，且该接口状态为“禁用”。
- **激活：**点击“禁用”状态接口操作列的【激活】图标，可将接口上线，上线后平台会继续记录该接口的调用记录，且该接口状态为“激活”。

## 调用记录监控

### **功能定位：**

接口调用的"调用日志"，仅监控状态为"正常使用中"的接口。

**ℹ️ 特别说明：** **调用记录数据来源于业务系统上报**。本模块仅提供记录的存储、查询和分析展示功能，不代表本模块直接发起了调用。

**操作步骤：**

1. 在接口集成模块页面，点击**【调用记录】**标签页。
2. 列表将显示所有的接口调用历史记录。

![image-20260123135731382](https://raw.githubusercontent.com/pinganki37-arch/Intelligent-Digital-Employee-Platform/main/assets/image-20260123135731382.png)

### 调⽤⽇志查询

记录所有接⼝调⽤的详细⽇志（时间、 参数、 结果、 状态） 。 ⽀持按时间、 接⼝、 状态查询。 ⽇志保留⼀个⽉， ⽀持导出。

**筛选调用记录：**

当记录较多时，可以使用筛选功能快速定位：

1. 使用页面上方的筛选条件栏：
   - **状态：** 选择“成功”或“失败”进行过滤。
   - **时间范围：** 选择“调用时间”的开始和结束时间点。
   - **编号筛选：** 输入调用ID或接口编码进行精确查询。
2. 点击**【查询】**按钮执行筛选。
3. 如需清除条件，点击**【重置】**按钮。

**查看调用详情：**

如果需要排查具体某次调用的详细信息：

1. 在调用记录列表中，找到目标记录，点击操作列的**【详情】**按钮。
2. 在弹出的【调用记录详情】窗口中，您可以查看：
   - **基本信息：** 接口名称、方法、状态、耗时等。
   - **请求信息：** 完整的请求URL、请求头（Headers）、请求参数（Query/Body）。
   - **响应信息：** 响应头、响应体（Response Body）。

## 使用说明文档

### 添加接口文档

本模块支持为注册的接口编写说明文档，方便团队协作。

1. 在接口集成模块页面，点击**【使用说明】**标签页。
2. 点击右上角**【添加接口文档】**按钮。
3. 在弹出的编辑器中：
   - **Markdown编辑器：** 在左侧区域输入Markdown格式的内容。
   - **预览区域：** 右侧实时展示文档渲染效果。
4. 编辑完成后，点击**【保存】**按钮。
